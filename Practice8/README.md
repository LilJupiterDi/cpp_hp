# Practice 8

## 1. Какие преимущества предоставляют гибридные вычисления?

Гибридные вычисления позволяют использовать сильные стороны и CPU, и GPU одновременно. CPU хорошо справляется с управлением, сложной логикой и последовательными участками кода, а GPU эффективно обрабатывает большие массивы однотипных данных. Такое разделение даёт прирост производительности и позволяет лучше загружать все доступные вычислительные ресурсы, вместо того чтобы простаивал один из процессоров.

---

## 2. Как минимизировать накладные расходы при передаче данных между CPU и GPU?

Основной способ — сокращать количество передач данных и передавать их крупными блоками, а не по частям. Полезно хранить данные на GPU как можно дольше, выполняя над ними несколько операций подряд без возврата на CPU. Также помогает использование асинхронных копирований и перекрытие передачи данных с вычислениями, чтобы время копирования не простаивало впустую.

---

## 3. Какие задачи лучше выполнять на CPU, а какие — на GPU?

На CPU лучше выполнять задачи с большим количеством ветвлений, сложной логикой и зависимостями между шагами, где параллелизм ограничен. CPU также удобен для управления программой и подготовки данных.
GPU эффективнее использовать для массовых параллельных вычислений, таких как обработка массивов, линейная алгебра, фильтрация, редукции и сканирование, где тысячи потоков выполняют одинаковые операции над разными данными.

---

## 4. Как можно улучшить производительность гибридного приложения?

Производительность повышается за счёт грамотного распределения работы между CPU и GPU и уменьшения точек синхронизации между ними. Важно избегать лишних копирований данных и правильно подбирать размеры блоков и потоков на GPU. Дополнительно помогает профилирование приложения, которое позволяет выявить узкие места и понять, какие части кода действительно стоит переносить на GPU, а какие выгоднее оставить на CPU.

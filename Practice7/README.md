# Practice 7

## 1. В чём разница между редукцией и сканированием?

Редукция сводит набор данных к одному значению. Например, сумма, минимум или максимум элементов массива. В процессе редукции промежуточные результаты постепенно объединяются, а на выходе остаётся одно число.

Сканирование, или префиксная сумма, сохраняет информацию о каждом шаге. Результатом является массив, где каждый элемент равен сумме всех предыдущих элементов исходного массива. В отличие от редукции, здесь важно не только финальное значение, но и все промежуточные результаты.

## 2. Какие типы памяти CUDA используются для оптимизации редукции и сканирования?

Для оптимизации активно используется разделяемая память, так как она позволяет потокам внутри одного блока быстро обмениваться промежуточными результатами без обращения к медленной глобальной памяти.
Регистры применяются для хранения временных значений внутри потока.
Глобальная память используется для исходных данных и записи итогового результата, но число обращений к ней стараются минимизировать. В некоторых случаях также используется константная память для параметров, общих для всех потоков.

## 3. Как можно оптимизировать префиксную сумму на GPU?

Оптимизация достигается за счёт разбиения массива на блоки и выполнения сканирования внутри каждого блока параллельно с использованием разделяемой памяти. Затем вычисляются смещения между блоками, которые добавляются к локальным результатам.
Также важно уменьшать количество синхронизаций и избегать конфликтов банков разделяемой памяти. Эффективное распределение потоков и выравнивание доступа к памяти позволяет значительно ускорить вычисление префиксной суммы.

## 4. Приведите пример задачи, где применяется сканирование.

Сканирование широко используется при фильтрации данных, когда нужно определить новые позиции элементов после отбора по условию. Например, при удалении нулевых значений из массива сначала строится массив префиксных сумм, который показывает, куда должен быть записан каждый элемент.
Также префиксная сумма применяется в алгоритмах сортировки, построении гистограмм и обработке графов, где требуется быстро вычислять накопленные значения.

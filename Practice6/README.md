# Practice 6

1. Основные типы памяти в OpenCL
   В OpenCL память делится на несколько уровней. Глобальная память — самая большая, доступна всем рабочим элементам, но сравнительно медленная. Локальная память общая для рабочей группы, работает быстрее и используется для обмена данными между потоками внутри группы. Приватная память принадлежит одному рабочему элементу, обычно это регистры. Есть ещё константная память — по сути часть глобальной, но только для чтения и с оптимизацией под широковещательный доступ.

2. Настройка глобальной и локальной рабочей группы
   Глобальный размер определяет общее количество рабочих элементов, которые будут запущены. Локальный размер задаёт, сколько рабочих элементов объединяются в одну группу. Обычно глобальный размер делают кратным локальному, чтобы железо не простаивало. Локальный размер выбирают с учётом архитектуры устройства, объёма локальной памяти и характера задачи. Маленькие группы дают гибкость, большие — лучше загружают вычислительные блоки, но могут упираться в ресурсы.

3. Отличие OpenCL от CUDA
   CUDA привязана к видеокартам NVIDIA и тесно заточена под их архитектуру. OpenCL — кроссплатформенный стандарт, он может работать на GPU разных производителей, CPU и даже других ускорителях. В CUDA обычно проще получить максимум производительности за счёт глубокой интеграции с железом, а OpenCL требует больше ручной настройки, но даёт свободу выбора платформы.

4. Преимущества использования OpenCL
   Главное преимущество — переносимость. Один и тот же код можно запускать на разном оборудовании с минимальными изменениями. Это удобно для долгоживущих проектов и гетерогенных систем. Плюс OpenCL позволяет использовать вычислительные ресурсы CPU и GPU в рамках одной модели программирования, что даёт гибкость при распределении нагрузки.
